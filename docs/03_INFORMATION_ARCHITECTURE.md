## Work index `/work`

Card grid with filters only if they add value. V1 does not require filters.

## Case study template

- clear title
- one-sentence outcome
- role and timeframe
- context
- problem
- ownership
- approach
- result
- engineering lessons
- related technologies
- next case study

## Research page `/research/knownet`

- plain-language abstract
- research question
- system overview
- personal contribution
- result and recognition
- publication metadata
- links
- selected system diagram only when a public-safe image is available

## About `/about`

Use a personal but professional narrative:
- engineering background
- statistics and evaluation background
- how these perspectives affect technical decisions
- what kind of problems Youfu wants to work on next

Avoid childhood biography or generic hobby filler.

## Resume `/resume`

- readable HTML summary
- downloadable PDF
- print-friendly styling
- no phone number in the visible web page by default
