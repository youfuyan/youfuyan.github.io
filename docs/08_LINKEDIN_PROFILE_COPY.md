## About

I’m a software engineer at Amazon working on creator and content systems. My recent
work has ranged from moving an image-matting workflow off a third-party vendor to
an in-house AWS service, to launching a business-critical API in a new region and
building event-driven services used across multiple teams.

I tend to do my best work on ambiguous problems that sit between system design,
implementation, evaluation, and operations. I’m comfortable going from a design
doc and data model to production rollout, on-call ownership, and the fixes that
follow.

A few examples:

• Designed and drove an AWS-native background-removal migration plan for
approximately 1M uncached image requests per month, projected to reduce annual
run-rate by about $96K, or approximately 80%, versus the third-party API based on
current traffic and public SageMaker pricing.

• Built a benchmark harness and LLM-as-a-judge evaluation pipeline to compare image
quality across multiple model and service backends.

• Launched a business-critical content API in a new AWS region, rolled traffic from
1% to 100%, and handled a live certificate issue without customer impact.

• Built the first version of an event-driven creator-follow notification service,
including race-safe DynamoDB writes and retry handling.

• Co-authored KNOWNet, an LLM and knowledge-graph system recognized with an IEEE
VIS 2024 Honorable Mention.

My background spans computer science, statistics, and quantitative methods. That
combination has been useful when a problem needs both production engineering and
careful evaluation.

I’m most interested in software engineering roles focused on AI/ML systems, backend
platforms, distributed systems, or full-stack AI products where reliability and
measurable impact matter.
